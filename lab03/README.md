Описание: игрок находит жителя, с которым хочет поторговаться, потому что после вылазки в пещеру он нашёл энное количество изумрудов. Игрок предлагает жителю изумруды либо случайно бьёт его (бывает), а житель просто орёт о том, что получил какое-то событие, наблюдатели это улавливают и реагируют в зависимости от типа полученного ими события.
1. Субъект Villager:
Состояние: Хранит данные о количестве изумрудов и статусе получения урона удара от игрока.
Управление "подписчиками": содержит список объектов, реализующих интерфейс Observer, и предоставляет методы для их добавления (attach) и удаления (detach).
События: При вызове методов offerEmeralds, hit объект автоматически инициирует метод notify(), который оповещает всех зарегистрированных наблюдателей об изменении состояния.
2. Интерфейс Observer:
Определяет единый стандарт для всех реагирующих компонентов. Метод update(subject: Villager) позволяет наблюдателям получать ссылку на субъект для анализа его актуального состояния.
3. Конкретные наблюдатели:
TradeSlot: Отвечает за цены. Наблюдатель реагирует на изменения состояния игрока: например, при получении персонажем урона (isHit), класс пересчитывает множитель цены (priceMultiplier), наказывая игрока за агрессию.
Mood: Отвечает за репрезентацию, в том числе визуальную для gui-шки, настроения. В зависимости от данных, полученных от субъекта, класс меняет строковое значение и будет менятся картинка.
